Buffalo Ts-Htgl-R5 升级

发表于 2016-04-19   |   分类于 Work   |  

巴比禄r5更换硬盘阵列。
一台服务4年的NAS文件服务器最近容量不够，需要升级硬盘。将新硬盘放入阵列柜发现启动的时候NAS找不到引导文件。
界面显示TFTP MODE E06 Lost boot image server ip 192.168.11.1

之后我找了一台PC将本地网卡的tcp地址更改到11网段，试图连通NAS好将系统刷进去。却发现无论怎么ping都ping不通这台NAS.

打电话给官方求助，官方告知更换全新硬盘没有操作系统，操作系统是写入在硬盘里面的。需要将老的3块硬盘先装回去，在放入一个新的硬盘与老的3硬盘进行同步。以此类推，放入一个新的，拆下一个老的，直到4块新硬盘都同步成功。
这样我就很纳闷了。
原因:
1:新的硬盘是大容量的，之前的硬盘是小容量的，那我放进去同步系统岂不是会把新的硬盘按照小空间去划分的，那我还升个毛线啊。
2:这么多数据同步4次，一次大概需要一天，这就浪费了4天。
官方说这台机器只有这一种方法解决更换全部硬盘的问题。


没办法，只能求助万能的google大神。
不出意外找到关于恢复的方法。
我大概整理了一下如下:
1:下载SolarWinds TFTP SERVER 这个软件 ，这台NAS必须使用TFTP协议来引导系统
http://solarwinds-tftp-server.soft112.com/
2:下载引导文件
https://drive.google.com/folderview?id=0B_oFnRRMAwk0SkYtdGpSdTNKTk0&usp=sharing#list
这台型号在最下面
3:下载固件与管理软件http://buffalo.jp/support_ap/support/products/ts_htgl_r5.html这里的固件版本号1.35与1.33都要下。管理软件名字NAS Navigator2 (Windows)


开始升级:
1:将本地pc的网卡的ip地址更改为192.168.11.1 子网255.255.255.0 网关不用动
2:用网线将pc与NAS连接，重启NAS。
3:重启完成，NAS会报警滴滴滴的叫。将下载的引导文件解压放到磁盘的根目录下例如(E:\xxx)。打开
下载的Solarwind TFTP这个软件，点击file-configxxx将文件夹指向引导文件，点击start，开始监听。


等待NAS自动去pc端读取引导文件。这个过程2分钟左右。读取完成，NAS会停止报警。这时候NAS已经进入了紧急恢复模式。这个时候就可以ping通这台NAS了。

4:解压下载管理软件，找到打开Naixxx.exe这个管理程序。它会自动搜索到当前这台NAS，点击它，将它的IP地址设置成从dhcp获取。拔掉pc这头的网线，将其插入到交换机上，让dhcp给它分配一个IP。
5:解压两个下载的固件包，进入1.33版本的文件夹，找到一个TSUpdater.ini的配置文件，用记事本方式打开。找到最后一行
VersionCheck = 0 将默认的1更改为0，保存。
在进入1.35版本的，也要打开将其更改为0，保存。
6:关闭pc的杀毒软件与防火墙，找到1.33版本文件夹里面的TSUpdater.exe，鼠标右键使用管理员方式打开。然后点击update耐心等待20分钟左右，这个时候固件就被刷到硬盘里面的。
7:如果想升级到1.35，就进入1.35的文件夹找到TSUpdater.exe与上面的方法一样，等待20分钟，NAS自动重启以后就升级完成。


PS:如果上面的方法失败。
1:换台操作系统，我win7的没成功，最后虚拟机上用xp才ok。
2:在执行操作前一定要关闭防火墙与杀毒软件。
3:一定要按照步骤来，
A.先用TFTP服务将引导文件刷入NAS.刷入完成一定不要重启NAS。
B.更改配置文件为0，先update1.33版本在update1.35版本，如果先update1.35会报错。
